今天我基于 VitePress 文档实践了自定义主题，并把博客的样式从默认文档风格改成了更适合个人博客的布局。

## 一、这次新增/修改的主题文件

### 1) 主题入口
- `docs/.vitepress/theme/index.js`

作用：
- 将自定义布局组件注册为全站布局入口。
- 引入全局主题变量文件 `custom.css`，用于统一品牌色。

### 2) 全局布局分发
- `docs/.vitepress/theme/Layout.vue`

作用：
- 根据页面状态分发到不同视图：
  - `NotFound`：404 页面
  - `Home`：首页（`layout: home`）
  - `Page`：普通内容页
- 提供站点级容器 `site-shell`。

### 3) 首页布局
- `docs/.vitepress/theme/Home.vue`

作用：
- 实现 Hero 区块（标题、副标题、按钮）。
- 从 `posts.data.ts` 读取文章列表，渲染“最新日志”卡片。
- 提供摘要提取逻辑（优先 `description`，否则从源码提取文本）。

### 4) 普通页面布局
- `docs/.vitepress/theme/Page.vue`

作用：
- 统一内容页阅读宽度和间距。
- 在文章存在 `date` / `author` 时显示元信息。
- 对 h1/h2/list/code 的节奏做基础优化。

### 5) 404 页面
- `docs/.vitepress/theme/NotFound.vue`

作用：
- 自定义 404 文案和返回首页入口。

### 6) 全局主题色变量（本次新增）
- `docs/.vitepress/theme/custom.css`

作用：
- 覆盖 VitePress 品牌色变量，统一按钮、链接、强调色。
- 同时提供浅色/深色模式下的对应颜色。

### 7) 文章数据加载器
- `docs/posts.data.ts`

作用：
- 使用 `createContentLoader('posts/*.md')` 自动收集文章。
- 过滤草稿并按日期倒序输出，供首页最新日志模块使用。

## 二、本次主题颜色调整（重点）

你要求“避免蓝紫色”，本次已完成以下改色：

1. 将主题主色从蓝紫方向切换为 **暖色琥珀/橙棕**：
   - `--vp-c-brand-1: #b45309`
   - `--vp-c-brand-2: #92400e`
   - `--vp-c-brand-3: #78350f`

2. 深色模式使用更亮的琥珀色，保证可读性：
   - `--vp-c-brand-1: #f59e0b`
   - `--vp-c-brand-2: #d97706`
   - `--vp-c-brand-3: #b45309`

3. 首页 Hero 渐变从“蓝紫”改为“橙 + 绿”点缀：
   - `linear-gradient(120deg, #b45309 20%, #16a34a 85%)`

4. 404 返回链接改为使用品牌变量：
   - `color: var(--vp-c-brand-1)`

这样可以避免整体视觉偏冷偏科技蓝，风格更温暖、更接近个人博客表达。

## 三、布局与样式改造总结

1. 首页不再是纯文档流，改为“Hero + 最新日志卡片”结构。
2. 日志区支持响应式：桌面双列、移动端单列。
3. 卡片加入 hover 反馈（位移/边框/阴影），提升层级感。
4. 内容页统一阅读宽度，减轻“又散又空”的观感。
5. 元信息（日期/作者）展示逻辑更清晰。

---

## 四、这次实践的收获

1. VitePress 自定义主题的核心是：`Layout.vue` 做页面分发，具体页面组件承担结构与样式。
2. 主题色最好集中在变量层管理，不要把颜色散落在每个组件里。
3. 首页与文章页应使用不同视觉策略：
   - 首页强调信息架构与入口；
   - 文章页强调阅读体验与排版节奏。

后续可以继续做：
- 给日志卡片增加标签（tag）与阅读时长。
- 在首页增加”按年份归档”的快捷入口。
- 增加深色模式下卡片阴影和边框的专门微调。

## 五、后续完善（2026-04-24）

### 1) 日志详情页重构（Page.vue）

将普通内容页升级为”左侧固定侧边栏 + 右侧文章主体”布局：

- **左侧侧边栏**：`position: fixed`，列出所有日志，高亮当前文章，独立于正文滚动。
- **文章主体**：卡片化（圆角边框），包含标题、元信息（更新日期、字数、阅读时长）、正文。
- **上一篇 / 下一篇导航**：上一篇在左，下一篇在右，卡片式布局。
- **字数与阅读时长**：从文章源码提取，过滤 Markdown 标记后统计中文字符与英文单词。

### 2) 顶部导航栏固定

将 `Layout.vue` 的导航栏从 `position: sticky` 改为 `position: fixed`，滚动时始终置顶，`site-main` 增加 `padding-top: 64px` 补偿高度。

### 3) 修复构建错误

**错误**：`Cannot read properties of undefined (reading 'imports')`

**根因**：所有文章 `.md` 文件直接放在 `docs/` 根目录，而 `config.mts` 的侧边栏链接和 `posts.data.ts` 的 glob 均指向 `posts/` 子目录，导致 VitePress 找不到对应的 Rollup chunk。

**修复**：将所有文章文件移入 `docs/posts/`，并补充 `docs/posts/index.md`。

### 4) 首页文案更新

将 Hero 标题改为”博观而约取，厚积而薄发”，副标题更新为更简洁的个人介绍。

### 5) 其他修复

- `docs/posts/2026-04-20-vitepress-docsRead-02.md`：补全缺失的 `::: code-group` 关闭标记。
- `config.mts`：添加 `ignoreDeadLinks: true`，避免构建因死链中断。
- `posts.data.ts`：过滤无 `date` 字段的条目，避免侧边栏出现”未知日期”。
